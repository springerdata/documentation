    Introduction

    Welcome to the ListingLeopard API! You can use our API to access information about Amazon products.

    You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    import requests

headers = {
    'X-api-key': 'apikey',
}

response = requests.get('https://api.listingleopard.com', headers=headers)

    # With shell, you can just pass the correct header with each request
curl -X 'GET' \
  'https://api.listingleopard.com' \
  -H 'X-api-key: apikey'

    fetch('https://api.listingleopard.com', {
    headers: {
        'X-api-key': 'apikey'
    }
});

    You must replace apikey with your actual api key.

    ListingLeopard uses API keys to allow access to the API. You can register for an API key here.

    ListingLeopard expects the API key to be included in all API requests to the server. It should be passed in a header:

    X-api-key: apikey

    You must replace apikey with your actual api key.

    Files

    The Files endpoints allow you to upload CSV files containing ASINs to ListingLeopard. This allows you to retrieve product information for thousands of ASINs at once.

    Upload a File

    import requests

headers = {
    'accept': 'application/json',
    'X-api-key': 'apikey',
}

params = {
    'description': 'ExampleDescription',
    'unique_tracking_id': 'ExampleTrackingID',
}

files = {
    'file': open('asin_template.csv', 'rb'),
}

response = requests.post('https://api.listingleopard.com/files/upload', params=params, headers=headers, files=files)

    curl -X 'POST' \
  'https://api.listingleopard.com/files/upload?description=ExampleDescription&unique_tracking_id=ExampleTrackingID' \
  -H 'accept: application/json' \
  -H 'X-api-key: apikey' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@asin_template.csv'

    const form = new FormData();
form.append('file', File(['<data goes here>'], 'asin_template.csv'));

fetch('https://api.listingleopard.com/files/upload?description=ExampleDescription&unique_tracking_id=ExampleTrackingID', {
    method: 'POST',
    headers: {
        'accept': 'application/json',
        'X-api-key': 'apikey',
        'Content-Type': 'multipart/form-data'
    },
    body: form
});

    The above command returns JSON structured like this:

    {
  "success": true,
  "file_id": "9NI5HRB53GV3JYZU",
  "status": "available",
  "created_at": "2023-01-01T01:01:00.000000+00:00",
  "request_count": 0,
  "asin_count": 12500,
  "description": "ExampleDescription",
  "unique_tracking_id": "ExampleTrackingID"
}

    This endpoint allows you to upload a file. The maximum file length is 20,000 ASINs. Retrieve a template here. Currently support domains are : amazon.com.

    HTTP Request

    POST https://api.listingleopard.com/files/upload

    File Upload Parameters

    The following parameters can be used when uploading a file:

    Parameter Required Description
    description false A description of the file being uploaded. Must be a string.
    unique_tracking_id false A unique tracking ID for the file being uploaded. Must be a string with no more than 20 alphanumeric characters.
    Tip - many find it useful to use unique_tracking_id as a client identifier.

    List Files Parameters

    file_id false The ID of the file for which to retrieve file information. Will return 1 file.

    Triggers

    Use these endpoints to automate your data collection. Prior to creating a trigger, you must create a file using the files endpoint.

    Create a Trigger

    import requests

headers = {
    'accept': 'application/json',
    'X-api-key': 'apikey',
    'Content-Type': 'application/json',
}

json_data = {
    "file_id": "9NI5HRB53GV3JYZU",
    "schedule_type": "daily",
    "trigger_function": ["stock"],
    "unique_tracking_id": "CLIENT225",
    "schedule_hours": [6, 13, 16],
}


response = requests.post('https://api.listingleopard.com/triggers/create', headers=headers, json=json_data)

    curl -X 'POST' \
  'https://api.listingleopard.com/triggers/create' \
  -H 'accept: application/json' \
  -H 'X-api-key: apikey' \
  -H 'Content-Type: application/json' \
  -d '{
  "file_id": "9NI5HRB53GV3JYZU",
  "schedule_type": "daily",
  "trigger_function": ["stock"],
  "unique_tracking_id": "CLIENT225",
  "schedule_hours": [6, 13, 16],
}'

    fetch('https://api.listingleopard.com/triggers/create', {
    method: 'POST',
    headers: {
        'accept': 'application/json',
        'X-api-key': 'apikey',
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
    "file_id": "9NI5HRB53GV3JYZU",
    "schedule_type": "daily",
    "trigger_function": ["stock"],
    "unique_tracking_id": "CLIENT225",
    "schedule_hours": [6, 13, 16],
  }
)
});

    This trigger will execute 3 times per day, because 3 different hours were specified in the schedule_hours field.

    Trigger Parameters

    The following parameters are used when creating a trigger:

    Parameter Required Description
    file_id true The ID of the file for which you want to create a trigger.
    schedule_type true The type of schedule for the trigger. Must be one of 'daily', 'weekly', or 'monthly'.
    trigger_function true A list of trigger functions to activate. Must contain one or both of the values 'stock' and 'reviews'.
    unique_tracking_id false A unique tracking ID for the trigger you want to create. Must be a string with no more than 20 alphanumeric characters.
    schedule_hours true Array of integers representing the hours at which to activate the trigger.
    schedule_days_of_week false Array of integers representing the days of the week on which to activate the trigger. Required if schedule_type is weekly.
    schedule_days_of_month false Array of integers representing the days of the month on which to activate the trigger. Required if schedule_type is monthly.

    Example Schedule Parameters

    Parameter Required Possible Values Example
    schedule_hours true Array of integers between 0 and 23. [6, 13, 16] for 6am 1 pm, and 4pm.
    schedule_days_of_week false Array of integers between 0 and 6. [0, 3] for Sunday and Wednesday.
    schedule_days_of_month false Array of integers between 1 and 31. [7, 14] for the 7th and 14th of month.
    The timezone of execution is UTC. You can use a converter here.
